# Local Setup

## Prerequisites
- Node.js 20+ and npm
- A Supabase project — see [`SUPABASE_SETUP.md`](SUPABASE_SETUP.md) first
- (Optional, for AI features) An Anthropic API key and an embedding provider API key (OpenAI or Voyage)

## Steps

1. **Install dependencies**
   ```bash
   npm install
   ```

2. **Configure environment variables**
   ```bash
   cp .env.example .env.local
   ```
   Fill in the values from your Supabase project settings (`Project Settings → API`) and, if you want AI features
   working, your Anthropic/embedding provider keys. See the comments in `.env.example` for which variables are
   public (safe for the browser) vs. server-only secrets.

3. **Apply database migrations**
   Using the Supabase CLI (recommended):
   ```bash
   npx supabase login
   npx supabase link --project-ref <your-project-ref>
   npx supabase db push
   ```
   Or paste each file in `supabase/migrations/` (in numeric order) into the Supabase SQL Editor.

4. **(Optional but recommended) Regenerate types against your live schema**
   ```bash
   npx supabase gen types typescript --project-id <your-project-ref> > src/types/database.types.ts
   ```
   `src/types/database.types.ts` in this repo was generated by `scripts/gen-types.mjs` (a best-effort parser of the
   migration files, since no live project was available in the build environment) — regenerating from your real,
   deployed schema with the Supabase CLI is the source of truth going forward.

5. **Seed development data (optional, non-production only)**
   ```bash
   npm run seed
   ```
   Requires `SUPABASE_SERVICE_ROLE_KEY` in `.env.local`. Creates departments, subjects, one account per role, sample
   resources/quizzes/assignments/announcements, and prints the demo credentials to the console. Never run this
   against a production project.

6. **Run the app**
   ```bash
   npm run dev
   ```
   Visit http://localhost:3000. Register with an `@tcetmumbai.in` email, or log in with a seeded demo account.

7. **Verify**
   ```bash
   npm run typecheck
   npm run lint
   npm run test
   ```
